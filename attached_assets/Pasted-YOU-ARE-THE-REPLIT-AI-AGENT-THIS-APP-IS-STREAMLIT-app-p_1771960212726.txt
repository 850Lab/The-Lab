PART 6 — DYNAMIC 72-HOUR WAR ROOM PLAN (THE ROCKET)
Create /src/war_room_plan.py:

Plan must be:
- dynamic (only include actions that apply)
- war-room tone (decisive, tactical)
- action-based (buttons route to panels)
- compact (top 3 visible, phases collapsed)
- compliant (no deception, no premature CFPB)

Phases:
0–6 LOCK FIELD:
- Freeze bureaus (links)
- Freeze ChexSystems (if bank goal OR generally beneficial)
- Baseline snapshot saved
- Proof readiness (if certified mail requires)

6–24 DEPLOY STRIKE:
- Generate letters (if not done)
- Send Certified Mail (if not done) (routes to send_mail)
- Log tracking # (routes to documents)
- Start 30-day clock (routes to tracker)

24–48 OPTIMIZE:
- Utilization assault if high/critical (routes to documents or “balance update” input)
- Inquiry discipline + rate-shop window guidance (auto mission)
- Targeted goodwill / executive call scripts ONLY when appropriate (paid collections/goodwill-eligible); scripts must be factual and polite.

48–72 LOAD ESCALATION WEAPONS:
- Prepare MOV + CFPB draft + exec escalation templates (PREP ONLY)
- Day 28 alert setup
- Day 31 escalation authorization (only when clock >=31)

Output:
- risk_level (HIGH/MOD/CONTROLLED)
- primary_lever (UTILIZATION/DELETION/STABILITY/OPTIMIZATION)
- top3_actions
- phases dict
- notes + transparency line (e.g., “Disputes rarely resolve in 72 hours; fastest lever is utilization & pull timing.”)

Completion:
st.session_state["war_room_done"][action_id] = bool
Show progress X/Y.

Hybrid tone:
- If within 72h: War Room language
- If day 1–30 clock running: Monitoring tone on workspace
- If day >=31: Escalation tone + “Authorized” label
