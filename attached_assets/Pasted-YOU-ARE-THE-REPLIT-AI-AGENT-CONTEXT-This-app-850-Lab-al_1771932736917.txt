FEATURE A: 72-HOUR TACTICAL CREDIT COMMAND PLAN (FREE)
PURPOSE
Turn the analysis into an operational, time-sequenced plan based on the user’s report/violations. This is NOT generic advice. It’s a prioritized “what to do next” for 72 hours.

WHERE IT LIVES (UI PLACEMENT)
- Add a new section on the existing “Results / Errors Found” page (or wherever violations are shown), placed ABOVE the letters CTA.
- It should be collapsed by default on mobile with a “View 72-Hour Plan” toggle (so it stays scannable).
- It must be available without payment.

WHAT IT SHOWS (STRUCTURE)
1) “Situation Snapshot” (scan-friendly)
   - Total issues found (already exists)
   - 3 chips (computed):
     • High Impact Items: X
     • Score-Damaging Items: Y
     • Quick Wins: Z
   - One sentence: “This plan sequences your next 72 hours for maximum impact.”

2) Timeline Cards (Day 1 / Day 2 / Day 3)
Each day shows 2–4 action cards. Each action card must include:
   - Action title (short)
   - “Why this matters” (1 sentence)
   - “Do this next” (1 sentence)
   - OPTIONAL: “Call / Message Script” expandable (accordion)
   - OPTIONAL: “Don’t do this” warning (only when relevant)

HOW TO GENERATE THE PLAN (LOGIC)
- Use existing parsed data: violations list, category counts, selected items.
- Create deterministic rule-based mapping (no new external calls required):
  Examples:
  - If there are “Negative Marks”: include a goodwill/exec-office call script action (Day 1).
  - If there are “Wrong Details” or “Inconsistencies”: include “Dispute under FCRA 611 for inaccurate/incomplete reporting” action (Day 1 or Day 2).
  - If utilization appears high (if you have balance/limit info): include “pay down to under 30%” action (Day 1/2).
  - Always include “Do NOT apply for new credit for 45 days” action (Day 1).
- Keep it short. Users scan. Don’t exceed:
  - Day 1: max 4 actions
  - Day 2: max 3 actions
  - Day 3: max 3 actions

COPY REQUIREMENTS
Use “mission control” language, not legal jargon:
- Title: “Your 72-Hour Credit Command Plan”
- Day headings: “Day 1 (Today)”, “Day 2”, “Day 3”
- Short lines, not paragraphs.

DELIVERABLES
- New components:
  - <CreditCommandPlan plan={plan} />
  - <PlanDayCard day={...} />
  - <PlanActionCard action={...} />
- New helper:
  - buildCreditCommandPlan(inputData) → plan object
